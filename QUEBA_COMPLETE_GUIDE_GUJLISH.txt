9. TECHNICAL — AAGAL SU BANAVU
================================================================================

RAKHO (CURRENT STACK):
- React + Vite + TypeScript + Tailwind + shadcn
- TanStack Query (API mate ready)
- i18n EN/GU (already started)

JOIYE (ARCHITECTURE):

  Patient PWA  ----->  Backend API  ----->  Postgres + Redis
  Clinic PWA   <-----  (Node/Nest)  <-----  Realtime (WebSocket)
                              |
                       SMS / WhatsApp

CORE DATA MODELS:
- Clinic (id, name, slug, timings, plan)
- Doctor (id, clinic_id, name)
- Patient (id, mobile, name)
- QueueSession (id, doctor_id, date, status: open|paused|closed)
- Token (id, session_id, patient_id, number,
         status: waiting|serving|skipped|completed|cancelled)
- DeviceAuth (optional — pachi)

CRITICAL PRODUCT RULES:
1. Token number SEQUENTIAL per doctor per day — Math.random() NATHI
2. Only one "serving" token at a time
3. Skip -> skipped list, can recall later
4. Complete -> increment current
5. Real-time: WebSocket or Supabase — patient screen refresh vagar update
6. OTP via MSG91 / Twilio / Fast2SMS (India)

/LOGIN PAGE NU CONFUSION:
Tyare "device token" (32 character string) ane "queue token" (#12) mix thay che.

MVP MA:
- Patient: phone OTP only (session cookie/JWT)
- Clinic staff: phone OTP ke PIN
- 32-char device token patient UX mathi kadhi nakhvo

LoginDialog (phone -> OTP -> name -> #token) — aa j sahi flow, RAKHO.

HARDCODED THINGS TO REMOVE:
- "Svedna Clinic" -> clinic profile / subdomain

CODE MA SU SAHI CHE (RAKHO):
